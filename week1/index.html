Local Storage

Javascripts localStorage can be used to save key/value pairs in the web browser. The data is stored even if the browser is closed.
Syntax for saving is as follows: localStorage.setItem(“key”, “value”);
Values can then be accessed through grabbing the info via key
Syntax for this: var newVariable = localStorage.getItem(“key”);

Source: https://www.w3schools.com/jsref/prop_win_localstorage.asp

Mobile first tips

Mobile first = better functionality
Loading must be quick (less steps = better)
Reachability matters when designing for touch (Key items must be in easy to reach places)

Increase speed by reducing image usage, use svg/webfonts, optimize CSS and JS files.

Placement matters
(Easier access can lead to a higher perception of speed, and higher engagement / satisfaction.)
Fixing “good enough” is what makes a website great.

Tricks for input bars on mobile devices
Float labels can be useful because the labels are inside the input box, and moves out of the way but is still
visible when typed (also saves space on the screen)
Make checkboxes and input boxes bigger (easier to select with thumbs on mobile)
Don’t show fields that aren’t required. Have them appear corresponding to options selected
Enabling autocomplete helps make info easier to fill out
Have passwords toggle visibility (helps prevent errors when filling out on mobile)
Brad Frost credit card input (look it up!)
Bring numpad rather than mobile text. (easier to input numbers)
Linking phone numbers on website make it much easier for mobile devices
Mobile ads should be small and simple
Small ads could get bigger upon click.
Ads could be videos
Do not use pop ups on ads (very annoying)
